This synthetic paid article body should never be shipped to an unauthorized browser. It exists only to demonstrate the disease locally. A weak implementation sends the full member-only article inside hidden HTML and trusts CSS or JavaScript to hide it. That is the bug. Automated agents, browser extensions, cached copies, search tools, and accidental exports can still see content that the server should have withheld. The correct fix is server-side authorization: send the public preview first, then return the full body only after the request is authorized.
